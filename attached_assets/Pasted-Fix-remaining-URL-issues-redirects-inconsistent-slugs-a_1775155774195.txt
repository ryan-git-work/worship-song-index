Fix remaining URL issues: redirects, inconsistent slugs, and uppercase keys
The previous deploy fixed internal links and generated new slugified theme URLs, but there are three remaining issues that need to be cleaned up.
1. Inconsistent theme slug format — some use underscores, some use hyphens
The sitemap currently has BOTH formats for multi-word themes:

great_commission (underscore)
body-of-christ (hyphen)
There are even duplicates like both greatness_of_god AND greatness-of-god

Also, the page title renders ugly for underscore slugs — it shows "Theme: Great_commission" with a literal underscore.
Fix needed: Standardize ALL theme slugs to use hyphens (not underscores). In getStaticPaths() for the theme page template, ensure every theme slug is generated with .replace(/\s+/g, '-').toLowerCase() or equivalent. The page title display should convert hyphens back to spaces with proper capitalization (e.g., "Great Commission" not "Great_commission"). Remove any duplicate entries from the sitemap.
2. Add 301 redirects for old broken URLs that Google has indexed
Google Search Console has 23 URLs it has been crawling that return 404. These are the OLD URL formats (with spaces, uppercase, and F%23). Since the pages now live at new slugified URLs, we need 301 redirects from the old URLs to the new ones.
For an Astro static site, the cleanest approach is to add redirect rules. In astro.config.mjs, add a redirects object, OR create a public/_redirects file (if deploying to Netlify/Cloudflare), OR handle it via middleware.

Ideally, rather than hard-coding each redirect, implement a middleware or catch-all approach:

Any /browse/themes/ URL with spaces → replace spaces with hyphens and redirect
Any /browse/keys/ URL with uppercase letters → lowercase and redirect
Any /browse/keys/ URL containing F%23 or F# → redirect to f-sharp

This way future URLs are also covered.
